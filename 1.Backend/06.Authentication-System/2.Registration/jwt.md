jwt (stateless authentication): user information + digital signature
jwt - composed of three parts
- header(contain metadata)+ playload(contain user data) + signature
signature is generated by : header+playload+secret key (using a cryptography algorithm to detect tempering)


jwt detect tempering: if any changes even single letter in playload or in headers the signature becomes invalid and token failed.



note: JWT does not hide the data.
JWT does not stop modifications.
JWT proves whether the data was modified.
